## Key Findings

### Experiment 7 & Bonus A: Matrix p-1 via Berggren
The order of B2 = [[2,1],[1,0]] mod p divides p^2-1 = (p-1)(p+1).
This means a SINGLE matrix-powering computation simultaneously tests
both p-1 smoothness AND p+1 smoothness. This is equivalent to a
combined Williams p+1 / Pollard p-1 method.

### Experiment 8: Gaussian Rho
Pollard rho in Z[i] mod N. The group Z[i]/(p) has order:
- p^2-1 if p = 3 mod 4 (p inert)
- (p-1)^2 if p = 1 mod 4 (p splits)
Cycle length ~ sqrt(group order). For p=3 mod 4, this is O(p)
which is WORSE than standard rho O(sqrt(p)). For p=1 mod 4,
cycle length ~ p-1, also worse. Gaussian rho does NOT help.

### Experiment 10: Mini QS with Z[i] structure
Standard QS using only tree primes (p=1 mod 4) for factor base.
Missing half the primes means we need a larger sieve interval.
The Z[i] structure doesn't compensate. Net: no improvement.

### Experiments 3, 4, 6: Tree-based cycle detection
All reduce to Pollard-rho-like cycle detection in (Z/NZ)^2.
The Berggren walk is deterministic, not pseudorandom enough.
Success depends on cycle structure of the specific matrix used.

## Conclusion

**Promising approaches found:**
- Exp8: Gaussian Rho (Z[i] Pollard): factored up to 70b
- Exp10: Hybrid Tree-Sieve (mini QS): factored up to 60b
- Bonus A: Matrix p-1 (B2 Berggren): factored up to 80b
- Bonus B: Gaussian p-1 (Z[i] p-1): factored up to 80b

**Best theoretical insight:** Matrix p-1 (Exp 7/Bonus A) combines
p-1 and p+1 methods into one computation. This is a known technique
(equivalent to Lucas sequences / Williams p+1), but the Berggren
matrix interpretation is novel and could be extended to higher-order
recurrences using products of multiple Berggren matrices.
